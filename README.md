# pokefuta-map

## まず最初にやること（MVP開始手順）

「いきなり実装」ではなく、最初の1〜2週間で**土台を作る**のが成功しやすいです。
以下の順番で進めるのをおすすめします。

### 1. 要件を固定する（半日）
- 対応プラットフォーム: iOS / Android（React Native + Expo）
- 公開範囲: 全体公開
- 予算: 月額 2,000 円目安
- MVP機能:
  1) ログイン
  2) 写真投稿
  3) 全国達成率
  4) 都道府県達成率
  5) 思い出タイムライン

### 2. ポケふたマスターデータを作る（最優先）
最初にやるべき実務はここです。

- まずは手動で `master/pokefuta.csv` を作る
- 最低限の列:
  - `id`
  - `name`
  - `prefecture`
  - `city`
  - `lat`
  - `lng`
  - `installed_at`（不明なら空）
- 理由: このマスターがないと達成率が計算できないため

> 外部サイト連携は、利用規約・二次利用条件の確認後に実施。

### 3. AWS を最小構成で立てる（1日）
- Cognito（認証）
- API Gateway + Lambda（投稿API・集計API）
- DynamoDB（訪問記録）
- S3（写真保存）

この4つだけで MVP は成立します。

### 4. データモデルを先に確定する（半日）
例:
- `pokefuta_master`
- `visits`
- `users`
- `prefecture_progress`

最初にER/テーブル設計を決めると、後の作り直しが激減します。

### 5. 画面は3画面から開始（2〜3日）
- ログイン画面
- 投稿画面（写真 + コメント + 位置情報任意）
- ダッシュボード（全国達成率 + 都道府県達成率）

### 6. 不正対策は「段階導入」
- MVP: GPS任意、EXIFがあれば信頼度UP
- 次フェーズ: GPS付きのみ「公式達成」にカウント
